# AI Knowledge NAS Ideation

## Product family

[[neural-foundry|Neural Foundry]] is the parent product family. Core modules: [[neural-vault|Neural Vault]], [[neural-forge|Neural Forge]], and [[neural-lens|Neural Lens]].

Working ideation folder for Nick's smart AI knowledge base / AI NAS concept.

Files:

- `ideation-notes.md` — problem, solution direction, session notes.
- `business-context.md` — business goals, stakeholders, strategy.
- `tech-requirements.md` — candidate architecture and components.
- `project-assumptions.md` — assumptions and risks to validate.
- `requirements.md` — initial functional and non-functional requirements.
- `research-notes.md` — early tool landscape notes and sources.
- `dataset-governance.md` — required governance pipeline for curated training datasets, fine-tuning, and future specialist model creation.
- `semantic-layer-ontology.md` — living ontology, entity resolution-linked graph, business semantics, and decision logic layer.
- `headless-agent-review.md` — headless agent-first operation model and human-in-the-loop review/approval service.
- `visualization-framework.md` — optional declarative visualization framework, default Knowledge Explorer, and agent-configurable graph/KPI/scenario views.
- `agent-memory-context-engine.md` — V1 target for personal OpenClaw memory appliance + small-business knowledge base, agent memory scopes, native notes/tables, and AgentOS boundary.
